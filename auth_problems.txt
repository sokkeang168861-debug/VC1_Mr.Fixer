Authentication Logic Problems Report
======================================

1. Missing Frontend Validation (Login & Signup)
-----------------------------------------------
- The Login and Signup pages do not check if fields are empty before submitting to the server.
- This allows empty strings to be sent to the backend, which is why clicking "Log In" with nothing entered still triggers an API call.

2. Missing Backend Validation (Login & Signup)
----------------------------------------------
- The 'authController.js' does not verify if 'email', 'password', or other required fields are present and non-empty.
- In the 'register' function, it allows creating a user with an empty email and empty password.
- Because 'email' is marked as UNIQUE in the database, only one "empty" user can exist, but it still allows one person to register with nothing.

3. "Random Account" Login Issue
-------------------------------
- If a user has already registered with empty fields (email="" and password=""), any subsequent login attempt with empty fields will match that specific user.
- This gives the impression of logging into a "random" account, but it's actually logging into the account that has empty credentials in the database.

4. Database Schema Mismatch
---------------------------
- The 'database.sql' defines a 'name' column, but the 'User.createUser' model tries to insert into a 'full_name' column.
- The 'database.sql' is missing the 'phone' column which is being used in the code.
- This inconsistency can cause registration to fail or behave unexpectedly depending on the actual state of the database.

5. Missing Logout Logic
-----------------------
- There is no 'logout' route defined in 'authRoutes.js' or 'authController.js'.
- While frontend logout (removing the token) is standard, the backend has no way to invalidate or blacklist tokens if needed.

6. Security Risks
-----------------
- 'JWT_SECRET' is hardcoded as "your_secret_key" in 'authController.js', which is insecure for production.
- Passwords, even if hashed, should not be allowed to be empty.
