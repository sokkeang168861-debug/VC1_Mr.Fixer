# Mr. Fixer - Database Migration Guide (Knex.js)

This project now uses Knex.js for database migrations. This replaces the manual process of exporting and importing SQL files. Every developer on the team can now sync their database schema using simple terminal commands.

---

## 1. What is a Migration?
A migration is like "version control" for your database. It is a JavaScript file that describes changes to the database (creating tables, adding columns, etc.). 
- 'Up' function: Defines what happens when you run the migration (e.g., CREATE TABLE).
- 'Down' function: Defines how to undo the migration (e.g., DROP TABLE).

---

## 2. Installation (For New Developers)
When a new developer joins or pulls the latest code, they should run:

1. Navigate to the backend folder:
   cd backend

2. Install the necessary packages:
   npm install

---

## 3. How to Run Migrations
To sync your local database with the latest schema defined in the project:

1. Ensure your database is running (XAMPP/MySQL).
2. Run this command in the 'backend' directory:
   npx knex migrate:latest

This will check which migrations haven't been run yet and execute them in the correct order.

---

## 4. How to Rollback (Undo)
If you made a mistake or want to revert the last batch of changes:

Run this command in the 'backend' directory:
npx knex migrate:rollback

---

## 5. How to Create a New Migration
If you need to add a new table or modify an existing one:

1. Generate a new migration file:
   npx knex migrate:make name_of_migration

2. Open the new file in 'backend/migrations/'.
3. Write your schema changes in the 'up' and 'down' functions.
4. Run 'npx knex migrate:latest' to apply it.
5. Commit the new migration file to Git so your teammates can run it.

---

## 6. Configuration (knexfile.js)
The 'knexfile.js' in the backend folder contains the database connection settings.
- It is currently set to use port 3307 and database 'mr_fixer_db'.
- If your local setup is different (e.g., port 3306), update the 'connection' settings in this file or use environment variables (.env).

---

## Summary of Commands (Run in /backend)
- npx knex migrate:latest   -> Update database to newest version
- npx knex migrate:rollback -> Undo the last migration batch
- npx knex migrate:status   -> See which migrations have been run
- npx knex migrate:make <name> -> Create a new migration file
